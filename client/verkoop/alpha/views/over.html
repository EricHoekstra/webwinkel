Architectuur

  • Client ↔ Server ↔ Database
  • Model View Controller
  • AngularJS, NodeJS, CentOS
  • Minimale afhankelijkheid, maximale samenhang

Server

  • Geschreven in NodeJS, JavaScript
  • Biedt een REST-API aan op https://www.erichoekstra.com/webwinkel/api. Voorbeelden van resources: product, foto, merk, adres, …
  • Volgt het Model-View-Controller-patroon.
  • Script import-afbeeldingen.js voor het importeren van foto's. Dit script wordt uitgevoerd op de server in een terminalsessie.
  • Ondersteunt OpenID; voorlopig alleen Google als provider.
  • Geïnstalleerd als een CentOS service in init.d.

Database

  • Van de database (en de rest van de applicatie) is een ontwerp.
  • De Adres-tabel bevat alle adressen in Den Haag en Amsterdam tot op toevoeging. Dat zijn een kleine 800.000 adressen.
  • Van 3.746 merken zijn in totaal 451.821 producten vastgelegd.
  • De server bedient alleen producten met een foto, door middel van een join, dat zijn er 1.881. Totaal kent de database 2.329 foto's.

Clients

  • Geschreven in AngularJS, Bootstrap, JavaScript, dus met HTML5-mogelijkheden en de Flexbox uit CSS.
  • Volgt het Model-View-Controller-patroon en de andere patronen in AngularJS, zoals dependency injection.
  • Een client voor de klant.
  • Voor de bedrijfsleider.
  • Voor de ontwikkelaar. Deze client dient voor opbouw van de functionaliteit.

Apps
